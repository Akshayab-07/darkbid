## Proof Structure (Dev 3 Generated)

### Circuit: `bid_range.circom`

**What it Proves** (without revealing bid amount):
```
✓ bidAmount >= reservePrice
✓ hash(bidAmount + bidSecret) == commitmentHash
```

**Inputs to Circuit**:
- `bidAmount` (private) - User's actual bid, kept secret
- `bidSecret` (private) - Random nonce generated by frontend
- `reservePrice` (public) - Minimum acceptable bid
- `commitmentHash` (public) - Commitment that user provides

**Output**:
- `validAmountProof` - 1 if amount is valid, 0 otherwise
- `hashMatchProof` - 1 if hash matches, 0 otherwise

Both must be 1 for proof to be valid.

## Public Signals (What Contract Verifies)

```typescript
publicSignals = [
  "50",        // [0] = reservePrice (public input from auction)
  "0x123abc..." // [1] = commitmentHash (public input from bidder)
```

These are the values the proof was generated for. Contract must verify proof against these exact signals.

### 3. Commitment Verification

```rust
pub fn commit_bid(
    ctx: Context<CommitBid>,
    commitment_hash: [u8; 32],
    proof_array: Vec<String>,
    public_signals: Vec<String>,
) -> Result<()> {
    let auction = &mut ctx.accounts.auction;
    let bidder = &ctx.accounts.bidder;
    
    // ❌ WRONG - Don't do this
    // let commitment = hash(user_provided_bid);
    
    // ✅ RIGHT - Use what proves in the ZK proof
    // The commitment_hash parameter MUST be in public_signals[1]
    
    // STEP 1: Verify the ZK proof
    let is_valid = verify_groth16_proof(
        &proof_array,
        &public_signals,
    )?;
    require!(is_valid, AuctionError::InvalidProof);
    
    // STEP 2: Extract and verify public signals
    let reserve_price = parse_u64(&public_signals[0])?;
    let commitment_from_proof = public_signals[1].clone();
    
    // CRITICAL: Commitment in proof must match what user provides
    require!(
        commitment_from_proof == format!("{}", commitment_hash),
        AuctionError::CommitmentMismatch
    );
    
    // STEP 3: Double-check reserve price matches auction
    require!(
        reserve_price == auction.reserve_price,
        AuctionError::ReserveMismatch
    );
    
    // STEP 4: Store the commitment (hash only, never the actual bid)
    ctx.accounts.bid_account.commitment_hash = commitment_hash;
    ctx.accounts.bid_account.bidder = bidder.key();
    ctx.accounts.bid_account.timestamp = clock::Clock::get()?.unix_timestamp;
    
    // STEP 5: Lock escrow funds
    // (Implementation depends on your escrow design)
    
    Ok(())
}
```

### 4. Reveal Phase (NO Proof Needed)

```rust
pub fn reveal_bid(
    ctx: Context<RevealBid>,
    bid_amount: u64,      // PLAIN - no longer secret
    bid_secret: u64,      // PLAIN - no longer secret
) -> Result<()> {
    let auction = &ctx.accounts.auction;
    let bid = &ctx.accounts.bid_account;
    
    // STEP 1: Re-compute hash from revealed values
    let revealed_hash = hash(bid_amount, bid_secret);
    
    // STEP 2: Verify it matches stored commitment
    require!(
        revealed_hash == bid.commitment_hash,
        AuctionError::CommitmentMismatch
    );
    
    // STEP 3: Double-check amount >= reserve (ZK already proved this!)
    // This is just a safety check
    require!(
        bid_amount >= auction.reserve_price,
        AuctionError::BidBelowReserve
    );
    
    // STEP 4: Store for winner selection
    bid.revealed_amount = bid_amount;
    bid.revealed = true;
    
    Ok(())
}
```

## Common Mistakes to Avoid

### ❌ Don't:
- Verify proof with wrong public signals
- Mix up public vs private inputs
- Store actual bid amount (defeats ZK!
- Forget to verify commitment matches proof
- Use wrong hash function (must be Poseidon, same as circuit)

### ✅ Do:
- Verify proof BEFORE accepting bid
- Check public signals in correct order: [reserve, commitment]
- Store ONLY the hash (proof), not the bid
- Lock escrow before confirmation
- Double-check reserve on reveal phase
